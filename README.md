### **Test Plan for demoblaze.com** 
________________________________________
&nbsp;

#### 1. **Objective**  <br> 
<font size="2"> Verify that the core shopping features on demoblaze.com function correctly for a standard user  -- including login, product browsing, add to cart, and checkout. The goal is to ensure users can complete a purchase without errors. 

#### 2. **Scope** - what will be tested <br>
<font size="2"> User login & logout Product browsing Add to cart Remove from cart Checkout form Navigation links Responsive layout Form validation
 
#### 3. **Out of scope** - will NOT be tested <br>
<font size="2"> Payment processing Admin panel Email notifications Performance testing Security / penetration 

&nbsp;
________________________________________


&nbsp;
#### 4. **Test types** <br>
<font size="2"> Functional    UI / visual 
<font size="2"> Negative testing  ·  Usability
<font size="2"> Cross-browser (Chrome, Firefox)
 
#### 5. **Test environment** <br> 
<font size="2"> URL: demoblaze.com
<font size="2"> Browser: Chrome 124, Firefox 125
<font size="2"> Device: Desktop + Mobile (375px) 

#### 6. **Tools** <br>
<font size="2"> Test cases: Google Sheets
<font size="2"> Bug tracking: Jira (free tier)
<font size="2"> Screenshots: Snagit / Lightshot
 
#### 7. **Timeline** <br> 
<font size="2"> Test case writing: Day 1
<font size="2"> Test execution: Day 2
<font size="2"> Bug reporting + summary: Day 3 

&nbsp;

________________________________________

&nbsp;

#### 8. **Entry & exit criteria** <br>
<font size="2"> **Start testing when...** <br>
<font size="2"> Site is accessible · Test cases are written · Test account is created on demoblaze

<font size="2">    **Stop testing when...** <br>
<font size="2"> All 20 test cases executed · All bugs logged · Summary report written

#### 9. **Risks & assumptions** <br>
<font size="2"> Demo site may be unstable or reset periodically  ·  No real payment data will be used  ·  Testing is manual only (no automation)  ·  Site data may not persist between sessions
